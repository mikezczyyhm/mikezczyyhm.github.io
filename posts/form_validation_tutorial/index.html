Form Validation Tutorial: Validating Text Input and Numeric Input

2020-10-18
#Java  #Android 

Introduction

This tutorial will guide you through implementing form validation for two input fields: a text input and a numeric input. We'll use a Java-like language and focus on input sanitization and numerical range checking.

Setting Up Variables

Start by getting the input values from your form:


String numericInputString = numericInput.getText().toString();
String textInputString = textInput.getText().toString();
boolean isValid = true;

Validating Text Input

We'll check if the text input is empty or contains invalid characters:


if (TextUtils.isEmpty(textInputString) || !textInputString.matches(".*[a-zA-Z0-9].*")) {
    textInput.setError("Invalid Text Input");
    isValid = false;
} else {
    textInput.setError(null);
}

Validating Numeric Input

Next, we'll validate the numeric input:


if (isNullOrEmpty(input)) {
  inputField.setError("Invalid Input: Empty or null");
  isValid = false;
} else {
  try {
      value = Integer.parseInt(input);
      if (value < MIN_VALUE || value >= MAX_VALUE || !isRelativelyPrime(value, MAX_VALUE)) {
          inputField.setError("Invalid Value: Out of range or not relatively prime");
          isValid = false;
      }
  } catch (NumberFormatException e) {
      inputField.setError("Invalid Input: Not a number");
      isValid = false;
  }
}

Handling the Result

After validation, you can use the `isValid` boolean to determine if the form can be submitted or if you need to show error messages to the user.

Additional Notes

  • Ensure you implement or import the `TextUtils.isEmpty()` method.
  • This code assumes you have input fields named `numericInput` and `textInput`. Adjust these names according to your actual form structure.
  • Error handling is done by setting error messages on the input fields. You may need to adjust this based on your UI framework.

Conclusion

This tutorial covered basic form validation techniques, including checking for empty inputs, validating text against a pattern, parsing and validating numerical inputs, and performing specific mathematical checks (like the GCD test for coprimality). You can expand on this foundation to create more complex validation schemes as needed for your application.
